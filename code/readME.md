running the pipeline: 

** first make sure subject's data is in this format: 
- subjXX (i.e. 'bb01')
  - mriYY (i.e. 'mri3') 	
    - parfiles
        - contains condition parfiles (ie. run1_2TR.par) for each run as generated by the experiment code
    - functionals
        - contains the raw nifti file of each run, downloaded from flywheel
        - nifti files to be named by their corresponding run number (i.e. ‘run1.nii.gz’, r'un2.nii.gz')
    - Inplane (make sure it’s ‘Inplane’ and not ‘inplane’ because mrVista will only accept ‘Inplane’ as the folder name)
        - this is the folder that the mrVista inplane / hidden inplane GUI works from so it has all the things that we save while working with the GUI
        - save subject’s inplane nifti within this folder
     
** then run mrVista_bbfloc_init

** save the subruns (output parfiles and func niftis) in a new directory that follows the same structure!

then run mrVista_bbfloc_main
